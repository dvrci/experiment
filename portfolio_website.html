PORTFOLIO WEBSITE

Overview

After gaining some invaluable experience in UX and UI design, I felt the next natural step was to add to my skill set by learning the basics of front-end development. This project sharpened my attention to detail during the design and wireframing phase, enabled me to improve communication with front-end engineers and deliver responsive designs that were easy and realistic to implement.

Additionally I was able to build a UX portfolio from scratch, showcasing past showcase projects and new skills in HTML, CSS and Javascript.

Tools used

Atom, GitHub Desktop/online platform, Sketch, W3C Markup Validation Service, WebAIM Contrast Checker, Codepen.io, GoogleFonts API

Process and Insights

I started by doing desk research on UX portfolio best practices and took note what stood out to me personally. Generally I noticed that the content of UX/UI portfolios were quite similar, nearly all included basics such as past projects, an about section and contact details. What stood out was how creatively some translated these contents and were able to bring across their personality, which became a big focus for me.

This was one reason why I decided to re-do the first part of the course using my own designs, instead of using wireframes provided. I wanted my website to be personalized and stand-out. To gage whether the designs were manageable with the skills I would learn throughout the course I discussed these with my mentor (who said it was more challenging than the standard template, but doable)

Once getting reassurance from my mentor, I set out to define the visual part of my portfolio, deciding on colours, fonts, icons.

The next (and hardest) part was to translate the designs into a working website. I learned how to structure html pages and styled these using CSS and Javascript. The designs were very challenging, as there was a lot of change depending on the viewport. As a result I became very familiar with media queries. Another part of the course was to build a hamburger menu for mobile viewports. I decided to take this out, as I didn’t feel it was necessary for the three menu items.

Once I had an MVP, I did qualitative user tests with 5 testers, which gave me valuable feedback which was implemented as part of the course.

The final step was to make sure my code was free of errors, easy to read and passed basic accessibility tests. To do this I added code linters W3C code linter and Atom-beautify to my development environment, ran my code through the W3C validator and fixed all warning and errors, checked color contrasts to make sure they passed the AA contrast ratio and finally did cross-browser testing.

I commited all changes, did a final push to Github and got the website live by using a hosting provider.

Even during the course I noticed a big change in how I could brief my designs and communicate in day to day working life. The course has enabled me to improve my design skills and given me an online presence I am proud of. Of course my goal is to continue to update the portfolio with new projects...it’s a project that will never truly be “done” so to say...
